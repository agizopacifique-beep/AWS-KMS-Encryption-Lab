## What This Lab Is About

Most security conversations focus on keeping attackers
out of a network. This lab focuses on a different
question: what happens to the data if someone gets in
anyway? The answer is encryption and specifically how
AWS manages encryption keys so that even if someone
accesses a file or a volume they still cannot read it
without the right key.

The lab uses AWS Key Management Service to create a
customer managed key, encrypt an S3 object and an EBS
volume with it, watch every operation get logged in
CloudTrail, and then disable the key to see exactly
what breaks and why.

## What I Learned

AWS never lets the KMS key touch the data directly. KMS
generates a separate data key for each object or volume.
That data key does the encrypting. The KMS key encrypts
the data key. The master key never leaves KMS and the
data key exists in plaintext only for the moment it is
actively being used then it is deleted.

Disabling a key is immediate and total. When the key was
disabled the EC2 instance could not boot, the S3 object
could not be opened, and CloudTrail logged every failed
attempt in real time. The instance went from Pending to
Stopped in seconds.

Encrypting an existing EBS volume takes several steps.
You cannot just click encrypt on a running volume. You
have to stop the instance, detach the volume, snapshot
it, create a new encrypted volume from the snapshot and
swap it in.

## Key Terms

**Customer Managed Key** -- A KMS key you create and
control. You decide who can use it and you can disable
or delete it at any time.

**SSE-KMS** -- Server-Side Encryption with AWS KMS.
When you upload an object to S3, the service requests
a data key from KMS and uses it to encrypt the file
before storing it.

**Data Key** -- A key generated by KMS for a specific
encryption operation. The plaintext version is used
once and then deleted.

**Envelope Encryption** -- The two-layer system this
lab uses. A data key encrypts the actual data. The KMS
key encrypts the data key.

**CloudTrail Event History** -- A 90-day log of API
calls made in your account. Every KMS encrypt, decrypt
and key management action shows up here with full context.
